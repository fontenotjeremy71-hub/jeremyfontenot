pfSense segmentation

Dual-interface VM separates upstream and internal networks.

VM 100 connects its WAN adapter to vmbr0 and its LAN adapter to vmbr1. This supports the segmented topology and routing role. Direct WAN-side RDP forwarding was later removed and replaced with authenticated VPN-only RDP.

VLAN 30 Linux Lab

Linux01 is segmented without breaking required identity services.

Linux01 was moved from earlier 10.10.20.x implementation states to VLAN 30 on 10.10.30.0/24. Proxmox uses VLAN-aware vmbr1, Linux01's virtual NIC is tagged VLAN 30, and pfSense routes the LinuxLab interface as vtnet1.30 with gateway 10.10.30.1/24.

Allowed services

Least-privilege access to DC01

OpenVPN remote access

Limitations

The validation boundary is part of the result.

Scope

Single host and single domain controller

The lab still has one physical Proxmox VE host and one domain controller. A second domain controller on the same host would not provide true physical resilience.

Network boundary

Private-upstream VPN test

OpenVPN was validated through pfSense WAN on the private upstream network, not through a proven public-Internet path.

Operations boundary

Read-only and target-local evidence limits

TCP reachability is not complete service validation, raw private inventories are not public, and retained isolated startup restore checks do not prove recurring disaster-recovery assurance, RTO, or RPO.

Future work

Broader logging and backup resilience

pfSense centralized logging is validated; broader Windows and Proxmox log forwarding and off-host or off-site backup resilience remain legitimate improvements.

Next steps

Strengthen recovery, access, and operational depth.

  1. 01Broaden centralized logging to Windows, Proxmox VE, and additional Linux sources after separate validation.
  2. 02Test OpenVPN reachability from a true public-Internet path if remote access must be proven beyond the private upstream household network.
  3. 03Improve off-host or off-site backup resilience.
  4. 04Add physical resilience before treating a second domain controller as high availability.
