Classifications and limitations

Repository structure validation: Tested / Passed. Baseline framework validation: Tested / Passed with expected negative connectivity case. Portfolio drift analysis: Tested / Partial match / Unresolved evidence gaps. Unresolved drift items are documented gaps, not failures.

Current evidence validates the captured repository and test state only. Timeout-handling validation does not prove service availability, and drift analysis does not by itself prove infrastructure failure.

Linux01 SSSD remediation

Responder activation conflict corrected with a rollback-first change.

The initial classification was SSSD responder activation conflict. Direct responders were configured, three responder sockets were enabled and failed, and config_file_version = 2 was rejected by SSSD 2.12 validation.

Control

Backup and rollback first

A preserved backup and rollback path were created before changing Linux01. The public page does not expose the root backup path.

Change

Smallest justified change

Only the rejected directive was removed. Conflicting NSS, PAM, and PAC responder sockets were disabled, SSSD was restarted, and historical failed-unit state was cleared.

Validation

Identity and SSH passed

sssd.service was enabled and active, zero failed systemd units remained, SSSD configuration validation passed, AD identity lookup passed, and a new AD-authenticated SSH session passed.

Claim boundaries

The public proof is intentionally narrow.

Scope

Personal lab only

This is a personal, nonproduction home lab. No client, employer, production, business, enterprise-scale, or operational-impact claim is made.

Privacy

Raw private material excluded

Raw target-local inventories, raw event logs, complete SSSD configuration files, authentication attempt details, local user paths, ticket-cache identifiers, secrets, keys, and tokens are not public.

Testing boundary

Reachability is not full service validation

TCP reachability can support a narrow network observation, but it is not complete application, authentication, firewall-policy, or service-correctness validation.

Recovery boundary

One restore drill is not recurring DR assurance

The direct restore evidence supports one isolated Linux01 restore drill to temporary VM 401. It does not establish recurring disaster-recovery assurance, RTO, RPO, SLA, or production readiness.
